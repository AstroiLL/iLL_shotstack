## Why

Текущая реализация валидатора (`check.py` и `check_json.py`) ищет файлы ресурсов в текущей рабочей директории, игнорируя поле `resourcesDir` из JSON. Это приводит к ложным предупреждениям о ненайденных файлах. Также необходима поддержка workspace структуры, где каждый проект хранится в отдельной папке со своими скриптами, медиафайлами и выходными файлами.

## What Changes

1. **Исправить валидацию путей к ресурсам** в `check.py`:
   - Искать файлы относительно директории, где находится JSON файл
   - Использовать `resourcesDir` из JSON корректно
   - Убрать ложные предупреждения

2. **Добавить поддержку workspace структуры**:
   - Рабочая директория = директория JSON файла
   - `resourcesDir` указывается относительно рабочей директории
   - Все операции выполняются относительно рабочей директории

3. **Добавить настраиваемый выход для assemble.py**:
   - По умолчанию: папка `output/` в рабочей директории
   - Имя выходного файла = имя скрипта + `.mp4`
   - Автоматическая индексация при существовании файла (не перезаписывать!)
   - CLI флаг `-o, --output` для указания кастомного пути

4. **Обновить документацию** в AGENTS.md, README.md

## Capabilities

### New Capabilities
- `resources-dir-validation`: Корректная валидация путей к ресурсам относительно расположения JSON файла
- `workspace-structure`: Поддержка структуры проекта с отдельными папками для скриптов, медиа и выходных файлов
- `output-management`: Управление выходными файлами с автоматической индексацией

### Modified Capabilities
- *Нет изменений существующих спецификаций*

## Impact

**Affected files:**
- `check.py` - исправление логики валидации ресурсов
- `check_json.py` - передача правильного пути к рабочей директории
- `assemble.py` - добавление выходной папки и имени файла
- `fast_clip/assembler.py` - поддержка кастомного выходного пути
- `AGENTS.md` - документация новых флагов
- `README.md` - описание workspace структуры

**Breaking Changes:**
- Нет - все изменения совместимы с текущим поведением
