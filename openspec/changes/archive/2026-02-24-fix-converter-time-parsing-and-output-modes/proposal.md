## Why

Конвертер MD в JSON содержит критическую ошибку в парсинге времени: формат `MM:SS:mmm` (минуты:секунды:миллисекунды) в таблице некорректно преобразуется в значение `trim`, что приводит к неверному вырезанию фрагментов видео. Кроме того, отсутствуют режимы вывода для разных сценариев использования: интерактивный, отладочный (verbose) и автоматический (quiet).

## What Changes

1. **Исправление парсинга времени** в `convert_script.py`:
   - Поддержка формата `MM:SS:mmm` (например, `00:01:800` = 1.8 сек)
   - Корректное вычисление `trim` из диапазона времени в таблице

2. **Добавление режимов вывода** в `convert_script.py`:
   - **`-v, --verbose`**: построчный вывод конвертации с деталями
   - **`-q, --quiet`**: полное отключение вывода, только код возврата
   - **Обычный режим**: только итоговый результат (имя, ресурсы, количество клипов)

3. **Добавление режима `-q`** в `check.py`:
   - Полностью без вывода для автоматического запуска
   - Код возврата 0 при успехе, 1 при ошибках

4. **Обновление документации** в `AGENTS.md` с описанием новых режимов
